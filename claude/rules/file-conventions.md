# File Conventions

## 末尾改行（POSIX準拠）

すべてのテキストファイル（ソースコード、ドキュメント、設定ファイル）は、末尾に改行文字 `\n` を含めて保存する。

**Why:**

- POSIX は「行」を改行文字で終端される文字列と定義しており、末尾改行のないファイルの最終行は "不完全な行" 扱いとなる
- 実害の例:
  - `wc -l` が最終行をカウントしない
  - `while read line` ループが最終行をスキップする場合がある
  - `cat file1 file2` でファイル境界の行が連結される
  - `git diff` で `\ No newline at end of file` 警告が出て差分が読みにくくなる
- prettier、gofmt、rustfmt、black 等の主要フォーマッターはデフォルトで強制している

**How to apply:**

- 新規ファイル作成時は末尾に `\n` を必ず含める
- 既存ファイルを編集する際も、末尾改行が欠けていたら補う
- Markdown の「行末2スペース＋改行」ルールは行末の話であり、ファイル末尾改行とは別物
